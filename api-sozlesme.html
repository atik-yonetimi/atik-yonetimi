Backend API Sözleşmesi — Tek Kaynak (v1)

Bu doküman, backend kodundan çıkarılmış en güncel sözleşmedir. Mobil ve simülasyon bu sayfayı referans alır.

Proje: Akıllı Şehir Çoklu Atık Türü Yönetimi ve Rota Optimizasyonu
Owner: Mehmet Demir
Kaynak: Backend-DB/waste-management
Sunucu: varsayılan http://localhost:8080
Swagger: /swagger-ui.html

1) Genel kurallar

  • Content-Type: application/json
  • Zaman: tüm zamanlar UTC, ISO 8601 (ör. 2026-03-25T10:00:00Z)
  • Doluluk: fillPercent 0–100 aralığında
  • Konum: lat/lng WGS84; backend null gelirse hata riski vardır (lat/lng gönderin)
Not: Bu sözleşme backend kodundaki doğrulamalara göre çıkarılmıştır. Proje kuralı gerektiren ama kodda henüz zorunlu olmayan alanlar ayrıca not edildi.

2) Yetkilendirme

Login ile token alınır, diğer bazı endpoint’lerde header gerekir:

Authorization: Bearer <token>
Uygulama notu: Mevcut kodda geçersiz token için 401/403 yerine 400 dönüyor.

3) Hata formatı

Tüm hatalar tek formatta döner:

}

7) Telemetry

POST /telemetry/ingest

Request

{
  "items": [
    {
      "containerId": 1,
      "fillPercent": 78.5,
      "lat": 41.015,
      "lng": 28.984,
      "recordedAt": "2026-03-25T10:00:00Z"
    }
  ]
}

Validation

  • containerId: zorunlu
  • fillPercent: 0–100 arası
  • recordedAt: zorunlu
  • lat/lng: kodda zorunlu değil ama fiilen gerekli (null olursa hata riski)

Response

{
  "acceptedCount": 1
}
Not: containerId backend’de yoksa 404 döner ve batch başarısız olur.

9) Stops

PATCH /stops/{stopId}/status

Header: Authorization: Bearer <token>

Request

{
  "status": "ARRIVED",
  "skipReason": null
}

Kurallar (koddan)

  • PENDING → ARRIVED geçer
  • PENDING → DONE geçmez (DONE için önce collection gerekir)
  • PENDING → SKIPPED geçer
  • ARRIVED → DONE geçer (önce collection olmalı)
  • PENDING’e geri dönüş yok
Proje kuralı: SKIPPED için skipReason zorunlu (kodda zorunlu değil).
